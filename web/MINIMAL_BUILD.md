# Минимальная сборка AmneziaWG Web Interface

Специально оптимизированная версия веб-интерфейса для серверов с ограниченными ресурсами (1vCPU, 1GB RAM).

## 🚀 Быстрый старт

### Для слабых серверов (1vCPU, 1GB RAM):

```bash
cd web/

# Минимальная сборка (оптимизированная)
make web-build-minimal

# Запуск минимальной версии
make web-up-minimal
```

### Доступ к интерфейсу:
- **Веб-интерфейс:** http://localhost:3000
- **API:** http://localhost:8080

## 🔧 Оптимизации

### Frontend (React):
- ✅ Ограничение памяти Node.js: `--max-old-space-size=512`
- ✅ Отключены source maps
- ✅ Только production зависимости
- ✅ Минимальный nginx конфиг
- ✅ Отключены health checks
- ✅ Кеширование статики

### Backend (Go):
- ✅ Ограничение памяти Go: `GOMEMLIMIT=128MiB`
- ✅ Сборка на scratch образе
- ✅ Статическая линковка
- ✅ Оптимизация размера бинарника (`-w -s`)
- ✅ Минимальный уровень логирования

### Docker:
- ✅ Ограничения ресурсов для контейнеров
- ✅ Frontend: 128M RAM, 0.3 CPU
- ✅ Backend: 256M RAM, 0.5 CPU
- ✅ Увеличены интервалы health checks

## 📊 Сравнение ресурсов

| Версия | Frontend RAM | Backend RAM | CPU | Сборка | 
|--------|-------------|-------------|-----|--------|
| **Обычная** | ~512MB | ~256MB | 100% | ~15-20 мин |
| **Минимальная** | ~64-128MB | ~128-256MB | ~80% | ~8-12 мин |

## 🛠️ Доступные команды

```bash
# Сборка
make web-build-minimal      # Минимальная сборка образов

# Управление
make web-up-minimal         # Запустить минимальную версию
make web-down-minimal       # Остановить минимальную версию

# Для сравнения (обычные команды)
make web-build              # Обычная сборка
make web-up                 # Обычный запуск
make web-down               # Обычная остановка
```

## ⚠️ Ограничения минимальной версии

1. **Функциональность:** Полная, но без некоторых продвинутых функций мониторинга
2. **Health checks:** Отключены для экономии ресурсов
3. **Логирование:** Минимальное (только ошибки)
4. **Nginx:** Упрощенная конфигурация
5. **Отладка:** Ограниченные возможности

## 🔄 Переключение между версиями

### С обычной на минимальную:
```bash
make web-down               # Остановить обычную версию
make web-build-minimal      # Собрать минимальную версию
make web-up-minimal         # Запустить минимальную версию
```

### С минимальной на обычную:
```bash
make web-down-minimal       # Остановить минимальную версию
make web-build              # Собрать обычную версию
make web-up                 # Запустить обычную версию
```

## 💡 Советы по оптимизации

### Для сборки:
1. **Освободите место:** `docker system prune -a`
2. **Закройте лишние приложения** во время сборки
3. **Увеличьте swap** если возможно
4. **Используйте SSD** для Docker

### Для работы:
1. **Мониторьте память:** `docker stats`
2. **Проверяйте логи:** `make web-logs`
3. **Регулярная очистка:** `docker system prune`

## 🆘 Устранение проблем

### Сборка зависает:
```bash
# Остановите все контейнеры
docker stop $(docker ps -aq)

# Очистите систему
docker system prune -a

# Попробуйте снова
make web-build-minimal
```

### Недостаточно памяти:
```bash
# Проверьте использование
free -h
docker stats

# Создайте swap файл (если нет)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Контейнер не запускается:
```bash
# Проверьте логи
docker logs amneziawg-web-api
docker logs amneziawg-web-frontend

# Проверьте статус
make web-status
```
